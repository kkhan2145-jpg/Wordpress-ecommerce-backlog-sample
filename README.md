# Product Backlog & User Stories — WordPress E-Commerce Re-Platforming

## Context
Sample product backlog created for an enterprise client migrating an existing online store to a WordPress/WooCommerce platform. The backlog reflects how I structure epics, user stories, and acceptance criteria when owning product delivery as a Project Manager / Business Analyst.

## My Role
- Translated client business goals into a prioritized backlog
- Facilitated stakeholder workshops to capture requirements
- Wrote user stories and acceptance criteria for the development team
- Maintained sprint mapping and priority across an Agile/Scrum delivery cycle

## What's Included
- `backlog.csv` — full backlog with epics, user stories, acceptance criteria, priority, and sprint allocation
- `backlog.md` — readable version of the same backlog for quick browsing

## Structure

Each item in the backlog includes:

| Field | Description |
|---|---|
| ID | Unique identifier |
| Epic | High-level feature grouping |
| User Story | "As a [user], I want [goal], so that [benefit]" |
| Acceptance Criteria | Conditions for the story to be marked done |
| Priority | High / Medium / Low |
| Sprint | Planned delivery sprint |
| Status | To Do / In Progress / Done |

## Tools Used
JIRA, Confluence, Google Workspace — this sample is exported in a portable format (CSV/Markdown) for easy review outside of a live PM tool.
